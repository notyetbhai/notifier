# 🔐 Vault Scriptorium (Notifier)
**The Neon Scriptorium** — A highly secure, glassmorphism-styled credential management and vault application deployed on Vercel at [notifier.yetindustry.online](https://notifier.yetindustry.online).

---

## ✨ Features
- **Modern "Cyber-Editorial" UI:** Developed with Tailwind CSS emphasizing dark mode, asymmetrical layouts, and glass panel components.
- **Double Encryption Model:** 
  - Operator accounts are secured via **Bcrypt** hashing.
  - Stored vault credentials are deep-encrypted using **AES (Fernet)** cryptography.
- **Dynamic Expiry Timers:** Visual cues for credential rotation logic.
  - 🟢 **Safe:** > 7 Days Remaining 
  - 🟡 **Warning:** 1-7 Days
  - 🔴 **Critical:** < 1 Day
  - ⚪ **Expired:** 0 Days (Grayscale)
- **Zero-Trust Access:** Passwords remain masked as `••••••••••••••••` by default. They are only decrypted and exposed on-demand via the `/reveal` API endpoints.
- **Serverless PostgreSQL Ready:** Configured to instantly connect to Neon/Supabase databases via SQLAlchemy and `psycopg2-binary`.

---

## 🚀 Tech Stack
- **Backend:** Python + Flask
- **Database:** PostgreSQL (via SQLAlchemy ORM)
- **Frontend:** HTML5, Tailwind CSS, minimal Vanilla JavaScript
- **Deployment & Hosting:** Vercel (Serverless Functions) + Neon Tech database

## 🛠️ Local Development Setup
Want to run the vault on your local machine? Ensure you have Python installed.

1. **Clone the repository:**
   ```bash
   git clone https://github.com/notyetbhai/notifier.git
   cd notifier
   ```

2. **Create a virtual environment:**
   ```bash
   python -m venv venv
   # On Windows: venv\Scripts\activate
   # On MacOS/Linux: source venv/bin/activate
   ```

3. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Initialize your Local Database & Keys:**
   This will generate your `vault.db` SQLite file locally and print out a fresh AES encryption key for your environment.
   ```bash
   python init_db.py
   ```

5. **Start the local server:**
   ```bash
   python run_app.py
   ```
   *Visit `http://localhost:5000` to access the vault!*

## 🌐 Deploying to Vercel
This project includes pre-configured `vercel.json` and `api/index.py` files to easily deploy to Vercel's serverless edge.

1. **Add your Environment Variables in Vercel:**
   - `DATABASE_URL` (Your Neon or Postgres DB string. e.g. `postgresql://...`)
   - `VAULT_ENCRYPTION_KEY` (The AES Key generated by `init_db.py`)
   - `SECRET_KEY` (Any long random string for Flask Session Cookies)

2. Once deployed, the application seamlessly auto-seeds the very first time it loads by capturing your `operator_01` account directly to your fresh Postgres database!

> [!WARNING]
> This application is highly secure but relies on keeping your `VAULT_ENCRYPTION_KEY` strictly out of version control. Never commit this key to GitHub!
